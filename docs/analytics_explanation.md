# Why this is an analytics project

NeuroQuantAI is a compact analytics case study. It shows how raw inputs can be turned into a controlled experiment, checked for quality, measured with clear KPIs, compared across scenarios, and communicated with simple visuals.

## Skills shown

### 1. Problem framing
The project defines a measurable question: how do different parameter choices change the final metrics under the same synthetic input conditions?

### 2. Data validation
The script checks for required columns, missing values, and invalid values before calculating results.

### 3. Experimental comparison
The project compares multiple configurations instead of reporting only one result. This creates a simple parameter sweep.

### 4. KPI design
The output table summarizes total result, baseline result, volatility, ratio metrics, downside gap, active days, count of changes, and correlation to the baseline.

### 5. Baseline comparison
The script compares the primary output against a baseline curve so the result has context.

### 6. Reporting
The repository includes methodology notes, sample outputs, visual assets, and a clean README so a reviewer can understand the workflow quickly.

## Interview explanation

I built NeuroQuantAI as a Python analytics case study. The goal was to show a reproducible workflow: generate controlled synthetic data, validate inputs, run a parameter comparison, calculate KPI-style metrics, compare against a baseline, and present the results clearly.

## Resume relevance

This project demonstrates analyst habits that transfer to business analytics, operations analytics, healthcare operations, product analytics, and reporting workflows: data checks, reproducibility, structured outputs, metric design, comparison tables, charts, and documentation.
